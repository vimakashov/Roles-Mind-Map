# Статус персонажа «Умер» — дизайн

## Цель

Дать возможность отметить персонажа как умершего по ходу повествования. В окне
свойств персонажа (там же, где имя/фамилия/возраст) добавляется чекбокс «Умер».
Когда он включён, поверх круглого аватара персонажа рисуется чёрный крест (X),
а сам аватар слегка приглушается — статус читается и на майнд-карте, и в
превью-аватаре модального окна.

## Объём (scope)

- Крест (X) + приглушение отображаются **в двух местах**: на узле майнд-карты
  (основной вид) и поверх превью-аватара в модальном окне (живая обратная связь
  при переключении чекбокса).
- Поведение умершего персонажа во всём остальном не меняется: связи, позиция,
  редактирование, удаление — как обычно. Это чисто визуальный/статусный флаг.

## Модель данных (server)

Одно новое поле в модели `Character` (`server/prisma/schema.prisma`):

```prisma
deceased Boolean @default(false)
```

Поле проходит через существующий конвейер почти без дополнительного кода:

- `characterCreateSchema` и `characterUpdateSchema` (`server/src/schemas.ts`)
  получают `deceased: z.boolean().optional().default(false)`. Роуты в
  `server/src/routes/characters.ts` уже делают `...fields` →
  `prisma.character.create/update`, поэтому правки роутов не нужны.
- `getBookGraph` (`server/src/services/graph.ts`) уже спредит `...c` на каждый
  узел, так что `deceased` сериализуется в payload графа автоматически.
- Миграционных файлов в проекте нет: на старте `server.ts` выполняет
  `prisma db push --skip-generate --accept-data-loss`. Добавление столбца с
  `@default(false)` неразрушающее, отдельной нормализации не требуется
  (флаг `--accept-data-loss` уже передаётся и нужен для другого — добавления
  unique-индекса связей).

## Единый источник правды для оверлея

По образцу существующего `avatarSvgMarkup` добавляется функция
`deceasedOverlaySvg()` в `web/src/lib/avatarSvg.ts`. Она возвращает один
самостоятельный SVG (`viewBox="0 0 100 100"`, явный `xmlns`, как у
`avatarSvgMarkup`), содержащий **и приглушение, и крест**, чтобы оба пути
рендеринга были пиксельно идентичны:

- полупрозрачная серая «вуаль»-круг — это и есть «приглушение»
  (читается как выцветший аватар; настоящий grayscale на canvas Cytoscape
  всё равно недоступен, см. ниже);
- жирный чёрный крест (X), вписанный в круг, с тонким светлым контуром-ореолом
  для контраста на тёмных/пользовательских аватарах, скруглённые концы линий.

Значения по умолчанию (подбираются при реализации): толщина штрихов X ≈ 10
единиц, концы в районе вписанной окружности (cx/cy 50, r ≈ 40), штрих `#111`
поверх светлого ореола `#fff`; вуаль `rgba(120,120,120,0.35)`.

Как и схематичный аватар, функция принимает опциональный `{ sized?: boolean }`,
добавляющий явные `width`/`height` для использования в качестве
`data:image/svg+xml,` background-image на canvas (viewBox-only SVG не имеет
разрешимого внутреннего размера — см. существующую заметку в CLAUDE.md).

## Пути рендеринга

Это единственная нетривиальная часть.

### Модальное окно — компонент `Avatar` (`web/src/components/Avatar.tsx`)

- Добавить проп `deceased?: boolean`.
- Обернуть текущий аватар (и `<img>`-ветку, и схематичную `<span>`-ветку) во
  внешний контейнер `position: relative`. Когда `deceased` — поверх рендерится
  абсолютно спозиционированный слой с `deceasedOverlaySvg()` через
  `dangerouslySetInnerHTML` (одинаково работает для кастомного и схематичного
  аватара).
- `CharacterModal` передаёт `deceased` из локального состояния, поэтому крест в
  превью появляется/исчезает сразу при клике по чекбоксу.

### Canvas — `graphAdapter.ts` + `MindMap.tsx`

Используются **слоистые background-image** Cytoscape (поддерживаются: массивы
для `background-image`, `background-fit`, `background-width`,
`background-height`).

- `graphAdapter.toElements` добавляет в `data` узла поле `overlayUri` —
  `deceasedOverlaySvg({ sized: true })`, обёрнутый в
  `"data:image/svg+xml," + encodeURIComponent(...)` (как схематичная ветка),
  **только когда** `deceased === true`; иначе поле отсутствует/`null`.
- В стиле узла `MindMap.tsx` `background-image` становится функцией-маппером,
  возвращающей `[avatarUri]` либо `[avatarUri, overlayUri]` в зависимости от
  наличия `overlayUri`. `background-fit`/`background-width`/`background-height`
  задаются массивами соответствующей длины (оба слоя — cover, 100%/100%).
- Эффект in-place синхронизации в `MindMap.tsx` уже спредит все изменяемые
  поля `data`, поэтому переключение `deceased` у существующего узла обновляет
  оверлей без переинициализации Cytoscape (id-set не меняется).

> Примечание про приглушение на canvas: Cytoscape не умеет накладывать
> CSS-фильтр (`grayscale`) на background-image, поэтому «приглушение» делается
> полупрозрачной серой вуалью, **запечённой в тот же оверлейный SVG**. Это же
> даёт единый источник правды и пиксельное совпадение модалки и canvas.

## Веб-обвязка

`deceased` добавляется в:

- тип `Character` (`web/src/types.ts`) — `deceased?: boolean`;
- `CharacterInput` (`web/src/api/client.ts`) — `deceased: boolean`
  (отправляется в теле create/update);
- маппинг `initial` в `BookScreen.tsx` — `deceased: modal.character.deceased ?? false`;
- `CharacterModal.tsx` — локальное состояние `deceased`
  (`useState(initial?.deceased ?? false)`), элемент управления
  `FormControlLabel` + `Checkbox` с подписью «Умер», размещённый среди полей
  (после «Возраст»), и значение включается в объект, передаваемый в `onSubmit`.

## Тестирование

- **Server** (`server/test/api.test.ts`): создать персонажа с `deceased: true`
  и PATCH-нуть существующего в `deceased: true`/обратно; проверить, что поле
  сохраняется и присутствует в payload графа. Прогонять полный
  `npm run test --workspace server`.
- **Web**:
  - `graphAdapter` — узел с `deceased: true` даёт `overlayUri`
    (массив background-image из двух слоёв); без флага — один слой.
  - `Avatar` — оверлей-слой рендерится только при `deceased`.
  - `CharacterModal` — чекбокс переключает состояние и значение уходит в
    `onSubmit`.
  - Существующий тест `MindMap` (форсит null-renderer Cytoscape) остаётся
    зелёным с маппером-массивом.

## Вне объёма (YAGNI)

- Дата/обстоятельства смерти, любые текстовые поля — нет, только булев флаг.
- Фильтрация/скрытие умерших на карте, отдельная легенда — нет.
- Настоящий grayscale-фильтр на canvas — нет (вуаль в оверлее достаточна).
